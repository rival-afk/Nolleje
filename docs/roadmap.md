# Дорожная карта создания nolleje и изучения нужных технологий

# Неделя 1
# Цель: Настроить всё, чтобы не отвлекаться потом.
- [ ] Создать репо на гитхаб. Изучить базовые команды гит
- [ ] Установить Python, PostgreSQL, VSCode
- [ ] Создать локальную БД
- [ ] Нарисовать упрощенную схему БД:
    - [ ] users (аккаунты. логин, айди, пароль, роль, почта, номер телефона)
    - [ ] schools (айди, название)
    - [ ] classes (айди, номер, буква, учащиеся, айди школы)
    - [ ] homework (айди, содержание, тип, айди класса, айди предмета, айди учителя, время выполнения (по дефолту до следующего урока), статус)
    - [ ] news (либо через тг, либо айди, содержание)
    - [ ] grades (айди, айди ученика, айди дз/айди урока, комментарий)
- [ ] Написать SQL-скрипты для создания таблиц
# Неделя 2-3: Бэк на FastAPI + авторизация
# Цель: Создать API, которое умеет отдавать данные и проверять права.
    # Изучить:
    - [ ] FastAPI: основы, роутеры, Pydantic модели
    - [ ] JWT-авторизация: библиотека python-jose, хэширование паролей (passlib)
    - [ ] Подключение к PostgreSQL через asyncpg или sqlalchemy
    # Практика:
    - [ ] Создать ручки регистрации и логина (/auth/register, /auth/login), возвращающие JWT-токен.
    - [ ] Реализовать зависимость get_current_user, которая проверяет токен и возвращает пользователя.
    - [ ] Написать простые CRUD для:
        - [ ] получения списка предметов ученика (/students/me/subjects)
        - [ ] получения заданий по предмету (/homeworks?subject_id=...)
        - [ ] получения списка классов для учителя (/teachers/me/classes) — пока без сложной фильтрации.
        - [ ] выставления оценки (только для учителя) — ручка POST /grades.
    - [ ] Проверить всё через автоматическую документацию FastAPI (Swagger UI) по адресу http://localhost:8000/docs.
# Неделя 4-5: Фронтенд (веб)
# Цель: Сверстать адаптивные страницы и связать их с API.
    # Изучить:
    - [ ] HTML/CSS: продвинутый Flex/Grid, медиа-запросы.
    - [ ] JavaScript: Fetch API, async/await, обработка ошибок.
    - [ ] Chart.js или простой Canvas для рисования пятиугольника (можно заменить на 5 прогресс-баров).
    # Практика:
    - [ ] Сверстать главную страницу ученика (карточки предметов, расписание (заглушка), лента новостей (заглушка)).
    - [ ] Сделать модальное окно с деталями задания (текст, статусы). Статусы показывать как радиокнопки.
    - [ ] Сверстать страницу учителя: список классов в виде простых карточек, при клике — список учеников (имитация, без подгрузки).
    - [ ] С помощью Fetch подгружать реальные предметы ученика с API и отображать в карточках.
    - [ ] Реализовать логин-форму, после успеха сохранять токен в localStorage и перенаправлять на нужный дашборд (по роли).
    - [ ] На странице учителя сделать кнопку "Выставить оценку" — отправлять POST на API.
# Неделя 6: Мобильное приложение (Android)
# Цель: Сделать один экран для ученика, который получает данные с нашего API.
    # Изучить:
    - [ ] Kotlin: основы (классы, функции), корутины для асинхронности.
    - [ ] Retrofit: настройка клиента, отправка запросов.
    - [ ] RecyclerView: отображение списка.
    # Практика:
    - [ ] Создать проект в Android Studio.
    - [ ] Добавить зависимости: Retrofit, Gson, okhttp.
    - [ ] Написать простой экран логина (поля email/password, кнопка). При успехе сохранять токен.
    - [ ] После логина открывать экран со списком предметов ученика (данные с API).
    - [ ] При клике на предмет показывать Toast с количеством заданий (или простой диалог).
    - [ ] Убедиться, что приложение работает с API, запущенным локально (через 10.0.2.2) или через cloudflared.
